\section{Interface Signal Table}
%=============================================================================

This section enumerates every signal crossing a block boundary within the CIF.
Signals are grouped by boundary in pipeline order: AXI slave interface first,
then write path, then read path, then shared blocks, then the CIF--MC~Core
interface.

Handshake types used below:
\begin{itemize}
  \item \textbf{VALID/READY} --- standard AXI4 handshake; transfer occurs when both asserted.
  \item \textbf{VALID/READY (tied)} --- READY permanently high; transfer occurs on VALID.
  \item \textbf{Pulse} --- single-cycle strobe, no handshake.
  \item \textbf{Level} --- combinational or registered level signal, no handshake.
  \item \textbf{FIFO push/pop} --- implicit handshake via FIFO full/empty flags.
\end{itemize}

\section{Arbitration Policy}
%=============================================================================

\subsection{Overview}

Two blocks in CIF are shared across all $N\_CLIENTS = 32$ AXI clients: the
\textbf{Transaction Tag Allocator} and the \textbf{Burst Splitter}. Both sit on
the common request path after the per-path Request Validators. This section
defines how access to these shared resources is arbitrated.

No other block requires explicit arbitration. The AR and AW Metadata FIFOs are
partitioned statically (4~slots per client); the ROB, RTT, and Response Tracker
are indexed directly by \texttt{txn\_tag} and require no arbitration. The
completion path from MC~Core is a single bus fanned out to the ROB, RTT, and
Response Tracker simultaneously, with each block filtering on
\texttt{resp\_type} --- no arbiter required.

\subsection{Transaction Tag Allocator}

The Tag Allocator is a single shared instance serving both read and write
paths.

\subsubsection*{Arbitration Mechanism}

The Tag Allocator accepts exactly one allocation request per cycle.
Upstream priority is resolved as follows:

\begin{enumerate}
  \item \textbf{Write path} and \textbf{read path} requests arrive at the Tag
        Allocator from their respective Request Validators (or Error Handlers).
        In CIF~v1, write and read paths share the Burst Splitter sequentially
        (single pipeline); the upstream MUX feeding the Tag Allocator selects
        between write-path and read-path requests using \textbf{fixed
        priority, write $>$ read}. If both paths present a request in the same
        cycle, the write-path request is accepted; the read-path request is held
        for the next cycle.
  \item Within a single path, only one AXID is presented per cycle (the FIFO
        head entry). No further arbitration is required at this level.
\end{enumerate}

\subsubsection*{Per-AXID Stall}

Allocation for a given AXID is blocked when
\texttt{occupancy[AXID] == MAX\_OUTSTANDING} (4). The Stall Logic asserts
\texttt{STALL} combinationally to the upstream block for that specific AXID
only. Other AXIDs are unaffected. \texttt{STALL} is released in the cycle
after the occupancy counter decrements (i.e., cycle $N+1$ after
\texttt{WR\_FREE\_VALID} or \texttt{RD\_FREE\_VALID} fires for a tag belonging
to that AXID).

\subsubsection*{Error Path}

The Error Handler presents allocation requests using the same interface as the
Request Validator (\texttt{REQ\_VALID}, \texttt{AXID}). Error-path allocations
are subject to the same per-AXID stall and write/read priority rules. No
bypass or priority elevation is granted to error transactions.

\subsubsection*{Simultaneous Free and Allocate}

If \texttt{alloc\_en} and \texttt{dec\_occ} fire in the same cycle for the
same AXID (one slot freed while a new one is being allocated), the occupancy
counter holds constant (net-zero change). This is a valid steady-state
operating condition and requires no special handling.

\subsection{Burst Splitter}

The Burst Splitter is a single shared two-stage pipeline instance serving both
read and write paths across all clients.

\subsubsection*{Arbitration Mechanism}

Access to the Burst Splitter is implicitly arbitrated by the upstream MUX that
feeds the Tag Allocator. Since the Tag Allocator and Burst Splitter are driven
from the same upstream MUX with the same priority policy (write $>$ read,
one request per cycle), the Burst Splitter processes exactly one transaction
at a time.

\subsubsection*{Pipeline Stall Policy}

The Burst Splitter Emit FSM stalls Stage~1 (via \texttt{s1\_busy}) during
\textsc{EMIT~B} to prevent the next FIFO entry from being consumed while the
second sub-transaction of a split is being emitted. \texttt{s1\_busy} is
deasserted on the cycle the FSM returns to \textsc{IDLE}. Backpressure from
downstream (\texttt{pkt\_ready = 0}) holds the FSM in its current state;
\texttt{s1\_busy} remains asserted until \textsc{IDLE} is reached.

\subsubsection*{Tag Allocator Stall Folding}

If the Tag Allocator asserts \texttt{STALL} before Stage~1 can proceed, the
stall is folded into \texttt{s1\_busy}. Stage~1 does not advance until
\texttt{STALL} deasserts and a valid tag is available.

\subsubsection*{Throughput}

Under no-split, no-stall conditions the Burst Splitter achieves one
transaction per cycle. A row-boundary split costs one additional cycle per
transaction (EMIT~A + EMIT~B). WRAP bursts iterate over the pre-computed
\texttt{beat\_addrs[]} table; throughput is one beat per cycle per WRAP
address, plus one additional cycle if a row-boundary split is also required.

\section{Reset and Initialization Sequence}
%=============================================================================

\subsection{Overview}

CIF supports two reset modes:

\begin{itemize}
  \item \textbf{Power-on reset (POR):} Asserted by the FPGA fabric on
        configuration. All flip-flops and BRAM init values take effect. CIF
        remains in reset until the AXI interconnect and MC~Core are both ready.
  \item \textbf{Soft reset:} Driven by a register write from a management
        plane. Must drain the pipeline before asserting; returns CIF to the same
        state as POR without reconfiguring the FPGA.
\end{itemize}

All resets are \textbf{synchronous, active-high}, qualified on
\texttt{posedge clk}. There is no asynchronous reset path in CIF.

\subsection{Reset Signal}

\begin{tabularx}{\textwidth}{L{2.5cm} C{0.8cm} L{3cm} X}
\toprule
\textbf{Signal} & \textbf{W} & \textbf{Source} & \textbf{Description} \\
\midrule
\texttt{rst\_n} & 1 & FPGA top / mgmt plane & Active-low synchronous reset; asserted low to reset all CIF blocks \\
\bottomrule
\end{tabularx}

\subsection{Power-On Reset Sequence}

The following sequence describes the required initialization order.
Steps within the same numbered stage may proceed in parallel.

\begin{enumerate}
  \item \textbf{Assert \texttt{rst\_n = 0}.}
        All registered state is cleared synchronously on the next
        \texttt{posedge clk}:
        \begin{itemize}
          \item AR/AW Metadata FIFO and W Data FIFO: empty, \texttt{ARREADY / AWREADY / WREADY = 0}.
          \item Write and Read Packet FIFOs: empty.
          \item Tag Allocator: all 64 occupancy counters $\leftarrow$ 0; all 64 seqnum counters $\leftarrow$ 0.
          \item Response Tracker register file: all entries cleared (\texttt{cmp\_pkts = 0}, \texttt{err\_sticky = 0}, \texttt{exp\_pkts = 0}).
          \item Response Tracker head pointer array: all 64 entries $\leftarrow$ 0.
          \item RTT register file: all 256 entries cleared.
          \item ROB BRAM: all 64 slots \texttt{valid = 0}.
          \item ROB free list: all 64 bits $\leftarrow$ 1 (all slots free).
          \item ROB lookup table: all 256 entries $\leftarrow$ 0.
          \item ROB head pointer array: all 64 registers $\leftarrow$ 0.
          \item ROB occupancy counter $\leftarrow$ 0.
          \item Burst Splitter Emit FSM $\leftarrow$ \textsc{IDLE}; all internal state registers cleared.
          \item Request Validators: combinational; no state to clear.
          \item Error Handlers: no persistent state to clear.
          \item \texttt{RESP\_READY} driven high immediately (tied; no reset dependency).
        \end{itemize}

  \item \textbf{Wait for MC~Core ready.}
        CIF must not issue any packets until MC~Core signals it is ready to
        accept traffic. \texttt{PKT\_VALID} must be held low (Packet FIFOs are
        empty during reset; no additional gating required if FIFOs reset
        correctly).

  \item \textbf{Deassert \texttt{rst\_n = 1}.}
        On the first \texttt{posedge clk} after deassertion:
        \begin{itemize}
          \item \texttt{AWREADY} and \texttt{ARREADY} are driven based on
                per-client FIFO slot availability. Both are high immediately
                after reset since all FIFO slots are free.
          \item \texttt{WREADY} is driven high immediately (W Data FIFO empty).
          \item The Tag Allocator begins accepting allocation requests.
          \item The Burst Splitter Emit FSM begins processing from
                \textsc{IDLE}.
        \end{itemize}

  \item \textbf{AXI traffic may begin.}
        First valid AXI transaction is accepted on the cycle after
        \texttt{AWREADY / ARREADY} are high and the AXI master presents a
        valid request.
\end{enumerate}

\subsection{Soft Reset Sequence}

Soft reset must be preceded by a pipeline drain to avoid losing in-flight
transactions.

\begin{enumerate}
  \item \textbf{Quiesce upstream.}
        The management plane instructs all AXI masters to stop issuing new
        transactions. New \texttt{AWVALID} and \texttt{ARVALID} are withheld.

  \item \textbf{Drain the request pipeline.}
        Wait until:
        \begin{itemize}
          \item AR and AW Metadata FIFOs are empty.
          \item W Data FIFO is empty.
          \item Write and Read Packet FIFOs are empty.
          \item Burst Splitter Emit FSM is in \textsc{IDLE}.
          \item Tag Allocator: all 64 occupancy counters $= 0$ (no in-flight transactions).
        \end{itemize}

  \item \textbf{Drain the completion pipeline.}
        Wait until:
        \begin{itemize}
          \item ROB occupancy counter $= 0$ (all read completions have been
                emitted to Merge Logic).
          \item Response Tracker: all 256 register file entries clear
                (\texttt{cmp\_pkts == exp\_pkts} and freed, or all
                \texttt{exp\_pkts == 0}).
          \item RTT: all 256 entries clear.
          \item No pending \texttt{BRESP} or \texttt{RRESP} outstanding on the
                AXI B or R channels.
        \end{itemize}

  \item \textbf{Assert \texttt{rst\_n = 0}.}
        Proceed identically to steps 1--4 of the power-on reset sequence.

\end{enumerate}

\subsection{BRAM Initialization Note}

The ROB BRAM \texttt{valid} bits are the only BRAM state that must be
explicitly initialized. On the cycle \texttt{rst\_n} deasserts, all 64
\texttt{valid} bits must read as 0. This is achieved by:
\begin{itemize}
  \item Using a synchronous-reset flip-flop for each \texttt{valid} bit
        (implemented as distributed RAM or shadow registers alongside the BRAM
        data), \textbf{or}
  \item Performing a 64-cycle initialization write loop on the BRAM write port
        during reset, setting \texttt{valid = 0} at each address before
        \texttt{rst\_n} deasserts.
\end{itemize}
The second option requires holding \texttt{rst\_n} low for at least 64 cycles
after POR. This is the recommended approach for UltraScale BRAM inference.

\section{Error Propagation Timing}
%=============================================================================

\subsection{Overview}

Address errors detected by the Request Validator are propagated through the
normal CIF pipeline as NOP packets using \texttt{pkt\_type[1:0] = 10}
(\texttt{Err\_write}) or \texttt{11} (\texttt{Err\_read}). This section traces
the cycle-by-cycle timing of \texttt{pkt\_type} from error detection to AXI
error response, and identifies where \texttt{pkt\_type} is set, carried, and
consumed. Timing counts assume no stalls unless noted.

\subsection{Write Path Error Propagation}

\begin{center}
\begin{tabular}{clL{9cm}}
\toprule
\textbf{Cycle} & \textbf{Stage} & \textbf{Event} \\
\midrule
$N$   & Request Validator & Address Range Check evaluates \texttt{addr\_ok = 0} combinationally. \texttt{ERR\_VALID} asserted to Error Handler; \texttt{AXID} forwarded. AW Metadata FIFO entry is consumed (popped) in this cycle. \\
$N$   & Error Handler     & Error Handler sets \texttt{pkt\_type = Err\_write} and presents \texttt{REQ\_VALID} + \texttt{AXID} to Tag Allocator in the same cycle. \\
$N+1$ & Tag Allocator     & \texttt{alloc\_en} fires (assuming no \texttt{STALL}). \texttt{TXN\_TAG} and \texttt{TAG\_VALID} registered; valid on cycle $N+1$. Occupancy counter and seqnum counter update take effect on cycle $N+1$. \\
$N+1$ & Burst Splitter Stage~1 & \texttt{pkt\_type = Err\_write} enters Stage~1 alongside \texttt{TXN\_TAG}, \texttt{ADDR}, \texttt{AXLEN}, \texttt{AXSIZE}. Stage~1 computes split decision combinationally; pipeline register clocked on cycle $N+1$. \\
$N+2$ & Burst Splitter Stage~2 & Emit FSM enters \textsc{EMIT~A}. \texttt{pkt\_type} carried unchanged through Sub-txn~A mux. Both sub-transactions (if split) carry the same \texttt{pkt\_type = Err\_write}. \\
$N+2$ & Packet Formation Unit & Write PFU assembles 618-bit packet with \texttt{pkt\_type[1:0] = 10} in bits [1:0]. \texttt{data[511:0]} and \texttt{wrapper[63:0]} are taken from the W Data FIFO as-is; no drain or discard. \\
$N+2$ & Write Packet FIFO & Packet pushed into FIFO. \\
$N+2 + F_W$ & MC Core & \texttt{Err\_write} packet dequeued from Write Packet FIFO and issued to MC Core (\texttt{PKT\_VALID} asserted). $F_W$ = Write Packet FIFO occupancy delay (0 under no backpressure). $N+2 + F_W + L_{MC}$ & Response Tracker & MC~Core asserts \texttt{RESP\_VALID} with \texttt{resp\_type = 10}. Completion Update Logic increments \texttt{cmp\_pkts} and sets \texttt{err\_sticky = 1}. $N+3 + F_W + L_{MC}$ & B Response Generator & Emit Logic (registered) drives \texttt{BRESP\_VALID}, \texttt{BID}, and \texttt{BRESP = DECERR} to B Response Generator on the cycle after \texttt{txn\_done}. \texttt{BVALID} is asserted on the AXI B channel. \\
\bottomrule
\end{tabular}
\end{center}

$L_{MC}$ denotes MC~Core's internal latency from packet receipt to error
completion assertion. This is a MC~Core parameter and is not constrained by
CIF.

\textbf{Key invariant:} \texttt{pkt\_type} is set once (at the Error Handler
output, cycle $N$) and carried unchanged through the S1$\rightarrow$S2 pipeline
register and Sub-txn muxes. Neither the Burst Splitter nor the PFU inspect or
modify \texttt{pkt\_type}. Both sub-transactions of a split carry the same
\texttt{pkt\_type}.

\subsection{Split Transaction Error Propagation}

When an error transaction also requires a row-boundary split
(\texttt{split\_needed = 1}), both sub-transactions carry the same
\texttt{pkt\_type}. The implications are:

\begin{itemize}
  \item \textbf{Write path:} Both \texttt{Err\_write} packets are issued to
        MC~Core. MC~Core returns one error completion per packet. The Response
        Tracker increments \texttt{cmp\_pkts} on each completion; \texttt{err\_sticky}
        is set on the first. \texttt{txn\_done} fires only after both
        completions arrive (\texttt{cmp\_pkts == exp\_pkts}).
        \texttt{BRESP = DECERR} is emitted once, covering the entire
        transaction.
  \item \textbf{Read path:} Both \texttt{Err\_read} packets are issued to
        MC~Core. MC~Core returns one error completion per packet (no data).
        The RTT increments \texttt{cmp\_pkts} on each; \texttt{rob\_last}
        fires after the second. The ROB emits two slots (both with error
        status); Merge Logic forwards both to the R Response Generator.
        \texttt{RLAST} is asserted on the final beat; both beats carry
        \texttt{RRESP = DECERR}.
  \item \textbf{Cycle cost of split:} EMIT~B adds one cycle to the Burst
        Splitter stage (Stage~2 holds \texttt{s1\_busy} for one additional
        cycle). All downstream stages (PFU, FIFO, MC~Core, response path) are
        otherwise unaffected by the split.
\end{itemize}

\subsection{Err\_write W Beat Handling}

W beats for an \texttt{Err\_write} transaction flow through the W Data FIFO
and Write PFU without modification or draining. The \texttt{pkt\_type} field in
the assembled packet signals to MC~Core that the data is invalid; MC~Core
discards the payload. No special W-beat gating or bypass path exists in CIF.
The number of W beats consumed by the PFU is determined by \texttt{AWLEN},
consistent with the normal write path.
